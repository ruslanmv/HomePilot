# Studio Frontend Design Specification

## Overview

A modern, professional content creation studio with seamless mature content governance.
Think: **Figma meets Midjourney** - clean, spacious, with powerful controls that don't overwhelm.

---

## Design Principles

1. **Progressive Disclosure** - Simple by default, powerful when needed
2. **Visual Feedback** - Clear status indicators for content ratings
3. **Safe Defaults** - SFW mode default, mature requires explicit opt-in
4. **Non-Judgmental** - Professional UI, no shame-based warnings for legal adult content

---

## Color System

```
SFW Mode:
  Primary: #3B82F6 (Blue)
  Background: #F8FAFC (Light gray)
  Accent: #10B981 (Green - safe indicator)

Mature Mode:
  Primary: #8B5CF6 (Purple)
  Background: #1E1B2E (Dark purple-gray)
  Accent: #F59E0B (Amber - mature indicator)

Danger (Blocked):
  #EF4444 (Red) - Only for illegal content blocks
```

## Interaction Flows

### Flow 1: First-Time Mature Mode Enable

```
1. User clicks [MATURE] toggle in header
2. Modal appears with age verification
3. User checks "I am 18+" checkbox
4. User clicks [Enable Mature]
5. UI transitions to dark purple theme
6. NSFW models and presets become visible
7. Toast: "Mature mode enabled. Explicit content now allowed."
```

### Flow 2: Image Generation with Preset

```
1. User selects "Anime Fan Service (Adult)" preset
2. Sampler settings auto-populate
3. User enters prompt in text area
4. Real-time policy check shows: âœ“ Allowed
5. User clicks [Generate]
6. Loading spinner with progress
7. Image appears in preview
8. Metadata saved to gallery
```

### Flow 3: Policy Block (Illegal Content)

```
1. User enters prompt with blocked term
2. Real-time policy check shows: âœ— Blocked
3. Red warning: "Blocked by absolute safety baseline"
4. Generate button disabled
5. Suggestion: "Remove prohibited terms to continue"
```
