## Adding the 10 new personas to the live gallery

There are two backends; choose the one your deployment uses.

### A. Static GitHub Pages (default)

```bash
echo '-include Makefile.personas' >> Makefile
make install-personas
```

`make install-personas` clones `HomePilotAI/personas`, builds the
`.hpersona` packages with real SHA-256, and **merges** the additive
entries into `docs/registry.json` via
`scripts/merge_additive_registry.py`. Existing entries (Scarlett, Atlas,
any future community submission) are preserved byte-for-byte; only new
ids are appended. The asset folders `docs/packages/` and
`docs/previews/` get the same additive treatment.

### B. Worker / R2 backend

The live `gallery.html` may be configured (via `window.GALLERY_API`) to
load from a Worker or R2 bucket that aggregates community submissions.
In that case the static file is a fallback; updating it does NOT change
what live users see. Instead:

1. Run `make install-personas` once locally to produce real SHA-256 +
   the asset tree under `docs/packages/` and `docs/previews/`.
2. Upload `docs/personas-additive.json` to the Worker / R2 source as a
   merge slice (or paste its `items[]` into the Worker's data store).
3. Upload the contents of `docs/packages/` and `docs/previews/` (only
   the `<id>/<v>/...` directories listed in `personas-additive.json`)
   to the matching paths in R2.

Neither path overwrites the 14 community entries.

## Production safety contract

- `make install-personas` MERGES; it never deletes.
- `make uninstall-personas` removes ONLY the 10 additive ids; community
  entries (including Scarlett, Atlas, David Mercer, Diana Brooks, Elena
  Voss, Felix Navarro, Kai Tanaka, Luca Moretti, Marcus Chen, Maya
  Chen, Nora Whitfield, Priya Sharma, Raven Okafor, Soren Lindqvist)
  are preserved.
- `scripts/merge_additive_registry.py` is the only writer of the
  `items[]` array; review it (it's < 100 lines) before adopting.
- The static `docs/registry.json` is opt-in via `Makefile.personas` and
  defaults to the existing fallback content. Removing
  `Makefile.personas` reverts the operator's local environment without
  needing a git revert.

## Verifying integrity

After `make install-personas`, every additive persona in the relevant
registry carries `latest.sha256`. To verify a download:

```bash
shasum -a 256 ~/Downloads/persona.hpersona
# compare with the gallery card / registry sha256 entry
```

Placeholder digests (prefix `placeholder-`) appear before the first
real build runs. After `make install-personas` the digests become real.
